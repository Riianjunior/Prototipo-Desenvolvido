# Prototipo-Desenvolvido
O protótipo de média fidelidade representa um aplicativo mobile de mercearia, apresentando telas estruturadas, cores e tipografia definidas. Ele simula a navegação principal do sistema, permitindo visualizar produtos, adicionar itens ao carrinho e finalizar pedidos, oferecendo uma experiência clara, intuitiva e próxima de um sistema real.

 Relatório – Desenvolvimento do Protótipo de Média Fidelidade
 
1. Introdução
Este relatório descreve o desenvolvimento de um protótipo de média fidelidade para um
aplicativo mobile de mercearia. O objetivo desta etapa é aplicar a linguagem visual definida
anteriormente e simular, de forma mais realista, a experiência de uso do sistema, considerando
navegação, hierarquia visual e feedback ao usuário.

2. Objetivo do Protótipo
O protótipo de média fidelidade tem como finalidade validar o fluxo principal do aplicativo, avaliar
a clareza das telas e verificar se a interação proposta atende às expectativas do usuário. Nesta
fase, o design ainda não é final, mas já apresenta estrutura visual consistente e elementos
interativos.

3. Telas Desenvolvidas
Foram desenvolvidas quatro telas principais, representando o fluxo central do aplicativo:
1 Tela Inicial (Home): apresenta o aplicativo, categorias de produtos, campo de busca e
destaques.
2 Lista de Produtos: exibe os itens disponíveis, com imagem, nome, preço e opção de
adicionar ao carrinho.
3 Carrinho de Compras: permite revisar os produtos selecionados, ajustar quantidades e
visualizar o valor total.
4 Confirmação do Pedido: informa ao usuário que a ação foi concluída com sucesso.
5. Linguagem Visual
A linguagem visual foi definida para transmitir simplicidade, confiança e facilidade de uso. As
cores predominantes são verde, branco e cinza claro, com o uso pontual do laranja para ações
principais. A tipografia escolhida foi a fonte Poppins, por sua boa legibilidade e aparência
moderna.
6. Hierarquia Visual
A hierarquia visual foi aplicada por meio do uso de tamanhos de fonte distintos, contraste de
cores e espaçamento adequado entre os elementos. Informações e ações mais importantes,
como botões principais e valores totais, recebem maior destaque para facilitar a compreensão
rápida da interface.
7. Feedback Visual
O protótipo apresenta feedback visual para confirmar as ações do usuário, como alteração de
cor dos botões, atualização imediata do carrinho e tela de confirmação ao finalizar o pedido.
Esses elementos reforçam a sensação de controle e segurança durante o uso do aplicativo.
8. Fluxo de Navegação
O fluxo principal do aplicativo ocorre da seguinte forma: Tela Inicial, Lista de Produtos, Carrinho
de Compras e Confirmação do Pedido. A navegação foi planejada para ser simples e intuitiva,
permitindo que o usuário complete sua tarefa principal sem dificuldades.
9. Considerações Finais
O protótipo de média fidelidade atende aos requisitos propostos, apresentando telas
organizadas, linguagem visual consistente e interações compreensíveis. Ele serve como base
para testes de usabilidade e para a evolução futura do projeto para um protótipo de alta
fidelidade.
